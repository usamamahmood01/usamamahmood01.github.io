Improving and Analyzing Data for Machine Learning and AI Models (Fall 2022)

In this project, I improved the data by 5%, collected by A. I. Whoo and Tanvir Rahman on individuals' activities, genders, and ages. I selected a sample of 100 individuals from the original dataset and added more features and information on the individuals' appearance. I also wrote a Python script to identify and highlight duplicate entries, resulting in a more accurate and precise dataset. After applying hierarchical clustering, I found that the top and bottom colors of individuals' clothing and their accessories were the most important features. This work demonstrates the significance of clean and accurate data for the success of machine learning and AI models and the importance of thoroughly cleaning and organizing data before using it to train models.

Semantic Segmentation with fcn-resnet-50 on CIFAR100

I built a convolutional neural network (CNN) to classify images from the CIFAR100 dataset. The architecture of the CNN included three convolutional layers, two fully connected layers, and max-pooling layers, and the activation function used was the rectified linear unit (RELU). I chose a loss function and an optimizer and trained the CNN for 30 epochs, saving the best model that achieved the highest accuracy on the training set. I then loaded my best model and used it to predict images in the test set, reporting the testing accuracy on the model that I trained. Source Code.

Procedure

I started by making a class map with distinct colors for each class. The pre-trained FCN_resnet_50 model was fed the supplied photos. It returned the segmented image. The class map was then used to color the segmented image, using a distinct color for each class. The model's output was then iterated over a further 21 times to produce the feature maps. The feature maps were recorded in a list, which was then shown as a tiled grid. You can see what is happening inside the model and where the segmentation is headed from the feature map. If you look closely at the feature map, you'll see that in many photographs the segments are white while the background is dark. You may discover photographs with all the segments white on the feature map, followed by images with just one segment white. This indicates that the model identified and marked that section.

Auto Mosaicking of Images Using SIFT-Based Correspondence Detection

To perform auto mosaicking of two images, I analyzed the use of corner detectors to compute correspondences between the images and the possibility of automatically creating image mosaics. To do this, I applied the SIFT algorithm to compute point correspondences between two images using their feature (corner) points. This allowed me to create a mosaic by blending the two images together in a way that preserved the important features and details of each image. The resulting mosaic would be a composite of the two images, with the goal of creating a seamless and visually appealing image that captures the content of both images. Source Code.

Creating Mosaics with User-Defined Blend Boundaries

To create a mosaic using Laplacian plane based reconstruction, the user can select the blend boundaries of the images using the mouse. The left and right images will share a joint region, and each mosaic will be made up of two images from different cameras or viewpoints. The process involves blending the left image with the right image to create the final mosaic. Source Code.
